standards

Accessibility First

Every project I build targets WCAG 2.2 AA compliance as a baseline — not an afterthought. Here's what that looks like in practice.

Colour Contrast

All text meets the 4.5:1 minimum ratio for normal text, and 3:1 for large text and UI components. For projects that require it — such as public sector sites — I target the AAA standard of 7:1.

Keyboard Navigation

Every interactive element is fully reachable and operable with a keyboard alone, with visible focus indicators.

Screen Reader Ready

Semantic HTML, proper ARIA labels, skip links, and meaningful alt text — included by default on every project.

Reduced Motion

All animations respect prefers-reduced-motion — no exceptions.
